Download

Grab the latest module release, check your device qualifies, and install it.

Requirements

⚠ Root access is mandatory
This is not an APK patch or a mod — it's a system-level module that runs inside the Android OS itself. Without the below, it cannot be installed and will not do anything.
  • An Android device with root access
  • A Zygisk-compatible root solution, such as Magisk with Zygisk enabled, or KernelSU / KernelSU-Next with a Zygisk implementation (Zygisk Next, ReZygisk, etc.)
  • School of Chaos Classic (com.vnlentertainment.socclassic) installed, on a device with 16GB+ RAM

Get the module

The module is published as a GitHub release, versioned alongside the source.

⬇ Download socfix_module.zip All releases

Installation

  1. Download socfix_module.zip from the latest release.
  2. Install it via your root manager app's "install module from storage" option, or from a root shell: 
    ksud module install socfix_module.zip
  3. Reboot your device. Zygisk modules only take effect after a reboot, since they hook into the zygote process at boot time.
  4. Launch the game as normal.

It should work on any Zygisk-capable rooted device with 16GB+ RAM hitting the same crash, since the fix targets the app's process directly rather than anything device-specific — but it has only been verified on the configuration above.

Building from source

Requires the Android NDK (built and tested with NDK r27).

cd module/jni
ndk-build

This produces module/jni/libs/arm64-v8a/libsocfix.so. Copy it to module/package/zygisk/arm64-v8a.so, then zip the contents of module/package/ (i.e. module.prop and the zygisk/ folder) to produce an installable module.
