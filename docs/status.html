Workaround

Experimental module

A root-only Zygisk module disables ASLR for the game process, on the theory that keeping its heap address low and stable avoids the overflow described on Home. It showed promising results in testing — but it was never proven to fully eliminate the bug, and it requires root.

The ready-to-flash package is published as v0.3 for people who accept those tradeoffs.

One run still crashed with the mitigation confirmed active. Primarily tested on KernelSU + Pixel 9 Pro XL. Other OEMs where the crash is confirmed were never tested with this module.
What resolves this

What actually needs to happen

The real fix has to come from VNL Entertainment updating the game's Unity engine version. Later Unity releases rewrote the native allocator responsible for this bug to handle 64-bit address spaces correctly — which is exactly why this bug class doesn't exist in modern Unity builds.

How you can help

The more device diversity we can document, the stronger the case for VNL Entertainment to prioritize a fix. If you're hitting this crash:

  1. Grab a logcat capture around the crash (filtered to com.vnlentertainment.socclassic is fine).
  2. Share it via an issue or the Discord.
  3. Include your device model/manufacturer and RAM if you know it — and whether you tried the module.

No root or technical setup required just to report — the log alone is genuinely useful.
