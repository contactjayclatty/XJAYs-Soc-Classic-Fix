Status

No fix is currently distributed or recommended. Here's exactly why, and what's actually needed to resolve this.

⚠ No confirmed public fix
This project does not currently publish or recommend any workaround. See below for why the one that was explored isn't being distributed.

The experimental workaround

A root-only Zygisk module was built and tested: it disables ASLR (address space layout randomization) for just the game's process, on the theory that keeping its heap address low and stable would avoid the overflow described on the Home page.

It showed promising results in repeated testing — but:

  • In one specific run, the mitigation was confirmed active (logged and verified) and the game still crashed — so it was never proven to fully eliminate the bug, just reduce its frequency.
  • It was only ever tested on one device/root configuration (KernelSU on a Pixel 9 Pro XL).
  • Since then, the crash has been confirmed on device families this workaround was never tested on — see Confirmed across multiple devices.
  • It requires root and a Zygisk-compatible setup, which most affected players won't have.

For all those reasons, it isn't published as a release or actively recommended. The source is kept in the repo's archive/ folder for anyone who wants to inspect, build, or experiment with it themselves — entirely at their own risk, with the understanding that it is unverified beyond a single test device.

What actually needs to happen

The real fix has to come from VNL Entertainment updating the game's Unity engine version. Later Unity releases rewrote the native allocator responsible for this bug to handle 64-bit address spaces correctly — which is exactly why this bug class doesn't exist in modern Unity builds.

How you can help

The more device diversity we can document, the stronger the case for VNL Entertainment to prioritize a real fix. If you're hitting this crash:

  1. Grab a logcat capture around the crash (filtered to com.vnlentertainment.socclassic is fine).
  2. Share it via an issue or the Discord.
  3. Include your device model/manufacturer and RAM if you know it.

No root or technical setup is required just to report — the log alone is genuinely useful.
