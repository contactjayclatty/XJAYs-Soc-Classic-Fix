Frequently asked questions

Common questions about the crash, the investigation, and where things currently stand.

Is there a fix I can install right now?
No confirmed, guaranteed fix — upstream still needs a Unity update from VNL. There is an experimental root-only Zygisk module (v0.3) that disables ASLR for the game process. It may reduce crashes but was never proven to eliminate them (it still crashed once with the mitigation active). Root + Zygisk required. Grab it from Download; background on Status.
Is this only on Pixel phones, or 16GB RAM devices?
No — that was the original theory, since it was first identified on a 16GB Pixel 9 Pro XL. The identical crash signature has since been confirmed on a Transsion-family device and a Xiaomi (MIUI) device, neither with confirmed 16GB RAM. See confirmed devices. The more accurate framing is that this is tied to Android's ASLR entropy in general, not a strict RAM cutoff.
Why does this only happen sometimes, not every launch?
The root cause is tied to ASLR, which re-randomizes where the game's heap gets placed in memory on every launch. Sometimes it lands under the problematic address threshold (game runs fine), sometimes it doesn't (instant crash). It's not your device or your install — it's effectively random chance on every cold start.
What exactly is causing the crash?
The game's engine (Unity 2017.4.34f1) has a native memory allocator that packs its bookkeeping assuming heap addresses stay under ~16GB of address space. When Android's ASLR places the heap above that threshold, the allocator's packed pointer representation overflows and the app segfaults. Full technical detail is on Home.
Will the developers fix this properly?
That's up to VNL Entertainment. A permanent fix requires updating the game's Unity engine version, which is out of this project's control. A full technical write-up has been sent to them — see the crash report. If it's fixed upstream, this project's job is done.
Can I try the experimental workaround myself?
Yes, if you understand it's experimental and requires root + Zygisk. Grab the ready-to-flash zip from the Download page or the v0.3 release. Source is in archive/module/ if you want to rebuild with the NDK.
Does this modify the game's APK?
No. Nothing in this project modifies, redistributes, or cracks the game's APK or assets. The experimental Zygisk module only changes an OS-level process flag (ASLR) from outside the app for that one package name.
How can I help?
Report your device hitting this crash — model, manufacturer, RAM if known, and a logcat capture around the crash if you can get one. No root or technical setup needed. See Contact or the Discord. The more devices we can document, the stronger the case for VNL Entertainment to prioritize a fix.
