Artifact

Experimental module — v0.3

A Zygisk module that disables ASLR for com.vnlentertainment.socclassic only, so the game's heap is less likely to land above the Unity 2017.4 allocator's ~16GB address ceiling. See root cause for the full explanation.

Install

Requirements & steps

  • Rooted Android device
  • Zygisk-compatible root (KernelSU, or Magisk with Zygisk enabled)
  • arm64-v8a (64-bit) — matches the game's APK
  1. Download socfix_module.zip from the release above.
  2. Flash it in your root manager (Modules → Install from storage).
  3. Reboot.
  4. Launch School of Chaos Classic. Check logcat for SocFix if you want to confirm the hook ran.
Caveats

Read this before installing

Experimental — not a guaranteed fix
This is not recommended as a permanent solution. Upstream still needs to update Unity. Install at your own risk.
  • In one test run the mitigation was confirmed active and the game still crashed — so it may reduce frequency rather than eliminate the bug.
  • Primarily tested on one setup: KernelSU on a Pixel 9 Pro XL.
  • The crash is confirmed on device families this module was never tested on — see confirmed devices.
  • Requires root; most affected players won't have that path.

Source lives in archive/module/ if you want to inspect or rebuild it yourself.

What resolves this

What actually needs to happen

The real fix has to come from VNL Entertainment updating the game's Unity engine version. Later Unity releases rewrote the native allocator responsible for this bug to handle 64-bit address spaces correctly — which is exactly why this bug class doesn't exist in modern Unity builds.

Contribute

How you can help

The more device diversity we can document, the stronger the case for VNL Entertainment to prioritize a fix. If you're hitting this crash:

  1. Grab a logcat capture around the crash (filtered to com.vnlentertainment.socclassic is fine).
  2. Share it via an issue or the Discord.
  3. Include your device model/manufacturer and RAM if you know it — and whether you tried the module.

No root or technical setup required just to report — the log alone is genuinely useful.
